# Docs recipe

## What to find

`docs/`, README, getting-started guides, MDX content roots.

## Recipe-specific checks

1. **Positioning promise** — does the intro match landing page and `PRODUCT.md` problem statement?
2. **Audience tone** — written for the same person as your ICP, or for internal devs only?
3. **Feature coverage** — docs emphasize what you're focused on, or document everything including non-goals?
4. **Getting started** — can a new user reach first value following docs alone?
5. **Stale claims** — docs promise features or flows that code/conversations say changed?

## Common misalignments

- Docs lead with API reference when users need a conceptual overview
- Getting started assumes context interviews show new users don't have
- Documents capabilities listed under `What you're not doing`
