# Onboarding recipe

## What to find

Signup flow, welcome screens, first-run wizard, empty states, activation steps.

## Recipe-specific checks

1. **Activation path** — does the flow lead toward your north star metric or a vanity step?
2. **Time to value** — how many steps before the user sees the core promise?
3. **Friction vs. evidence** — steps that interviews flagged as drop-off points; are they still there?
4. **Defaults** — do pre-filled choices match your intended primary use case?
5. **Skip paths** — can users bypass setup that matters for activation?

## Common misalignments

- Onboarding optimizes for profile completion, not the behavior in `STRATEGY.md` focus
- Asks for data you don't use yet (trust tax)
- First screen copy doesn't match landing page promise (bait-and-switch feel)
